## About Me

I work with infrastructure that people usually remember only when it breaks.

Production incidents, strange network behavior, unstable routing, weird Docker problems, overloaded proxies, broken CI/CD, mail systems from hell — this is where I feel comfortable.

I design systems that should survive reality, not presentations.

That means:

- stable Linux infrastructure
- networking that behaves predictably
- automation that actually saves time
- monitoring that helps before disaster
- security without unnecessary theater

I prefer boring systems that work over beautiful systems that fail.
